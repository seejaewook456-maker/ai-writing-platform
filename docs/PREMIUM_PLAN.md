# Premium 플랜 정책 문서

> 이 문서는 Premium 구독 기능의 설계 정책을 정의합니다.
> 코드 변경과 함께 이 문서도 반드시 업데이트해야 합니다.

---

## 현재 상태 (2026-06-27)

- Premium 기능은 아직 구현되지 않았습니다.
- 모든 사용자는 FREE 플랜으로 가입하며, 현재 저장 제한이 없습니다.
- `User.plan` 필드는 존재하지만 저장 제한 로직은 미적용 상태입니다.
- 현재 목표는 **사용자 확보와 서비스 안정화**입니다.

---

## 플랜 정의

| 플랜 | 설명 | 저장 제한 | 현재 적용 여부 |
|---|---|---|---|
| `FREE` | 무료 플랜 | 향후 회차 30개, 인물 50명, 설정 100개 (예시) | 제한 없음 (미적용) |
| `PREMIUM` | 유료 플랜 | 높은 한도 또는 무제한 | 미구현 |

> 위 숫자는 예시이며 출시 전 변경될 수 있습니다.

---

## 기존 사용자 보호 정책 (불변 원칙)

Premium 출시 이후에도 기존 무료 사용자의 데이터는 **절대로 삭제하거나 숨기지 않습니다.**

### 예시 시나리오

- Premium 출시 전: 회차 120개 저장한 FREE 사용자
- Premium 출시 후 FREE 제한이 30개로 설정되더라도
  - 기존 120개 → **그대로 유지** (조회/수정/삭제 모두 가능)
  - **새로운 회차 생성만 제한**
  - 삭제하여 30개 이하가 되면 → 다시 생성 가능
  - PREMIUM 구독 시 → 제한 해제

이 정책은 변경하지 않는 것을 원칙으로 합니다.

---

## 저장 제한 적용 범위

저장 제한은 **생성(Create) 작업에서만 검사**합니다.

| 작업 | 제한 여부 |
|---|---|
| 생성 (Create) | 제한 검사 대상 |
| 조회 (Read) | 제한 없음 |
| 수정 (Update) | 제한 없음 |
| 삭제 (Delete) | 제한 없음 |

---

## 저장 제한 항목 (향후 구현 예정)

| 항목 | FREE 한도 | PREMIUM 한도 |
|---|---|---|
| 회차 (Episode) | 30개 (예시) | 무제한 또는 높은 한도 |
| 인물 (Character) | 50명 (예시) | 무제한 또는 높은 한도 |
| 세계관 설정 (WorldSetting) | 100개 (예시) | 무제한 또는 높은 한도 |

> 숫자는 예시이며 출시 전 확정됩니다.

## 설정 관리 원칙

저장 제한 숫자는 코드에 하드코딩하지 않습니다.
향후 아래 구조를 참고하여 설정 파일 또는 별도 테이블에서 관리합니다.

```yaml
# 예시 구조 (향후 application.yml 또는 DB 테이블로 관리)
plan:
  free:
    episode-limit: 30
    character-limit: 50
    world-setting-limit: 100
  premium:
    episode-limit: -1      # -1 = 무제한
    character-limit: -1
    world-setting-limit: -1
```

## DB 스키마

### users 테이블 추가 컬럼

| 컬럼 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `plan` | `VARCHAR(50)` | `FREE` | 사용자 플랜 |
| `plan_start_at` | `DATETIME` | - | 플랜 시작일 (향후 추가 예정) |
| `plan_end_at` | `DATETIME` | - | 플랜 만료일 (향후 추가 예정) |

> `plan_start_at`, `plan_end_at`은 Premium 결제 구현 시 추가합니다.

---

## 향후 Premium 구현 시 주의사항

1. **기존 사용자 데이터 보호**: FREE 제한 초과 데이터는 조회/수정/삭제만 허용, 신규 생성만 차단
2. **동시성 처리**: 생성 제한 검사는 DB 레벨 트랜잭션으로 처리하여 동시 요청 시 초과 저장 방지
3. **API 하위 호환성**: 기존 프론트엔드가 깨지지 않도록 plan 관련 응답 필드는 Optional 처리
4. **플랜 만료 처리**: `planEndAt` 초과 시 자동으로 FREE로 다운그레이드되는 스케줄러 필요
